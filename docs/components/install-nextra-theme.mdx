To support default search functionality, install **Pagefind** as a dev
dependency:

```sh npm2yarn
npm i -D pagefind
```

Add the following scripts to your `package.json`:

```json filename="package.json"
"scripts": {
  "dev": "next",
  "build": "next build",
  "start": "next start",
  "postbuild": "pagefind --site .next/server/app --output-path public/_pagefind"
},
```

> [!TIP]
>
> You can enable Turbopack in development by appending the `--turbopack` flag to
> the `dev` command:
>
> ```diff filename="package.json"
> - "dev": "next",
> + "dev": "next --turbopack",
> ```
>
> For Static Exports, adjust the postbuild script to
>
> ```diff filename="package.json"
> - "postbuild": "pagefind --site .next/server/app --output-path public/_pagefind",
> + "postbuild": "pagefind --site .next/server/app --output-path out/_pagefind"
> ```

You can start the server in development mode with the following command
according to your package manager:

```sh npm2yarn
npm run dev
```

or in production mode:

```sh npm2yarn
npm run build
npm run start
```

> [!NOTE]
>
> If you're not familiar with Next.js, note that development mode is
> significantly slower since Next.js compiles every page you navigate to.

### Ignore Pagefind generated files

Add `_pagefind/` to `.gitignore`. This prevents committing the generated search
index files, which are not necessary for version control.

```plaintext filename=".gitignore"
_pagefind/
```

### Add Next.js config

Create a `next.config.mjs` file in your project's root directory with the
following content:

```js filename="next.config.mjs"
import nextra from 'nextra'

const withNextra = nextra({
  // ... Other Nextra config options
})

// You can include other Next.js configuration options here, in addition to Nextra settings:
export default withNextra({
  // ... Other Next.js config options
})
```

With this configuration, Nextra will handle Markdown files in your Next.js
project. For more Nextra configuration options, check out the
[Guide](/docs/guide).
