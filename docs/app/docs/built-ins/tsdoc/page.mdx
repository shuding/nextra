# TSDoc Component

Component to generate documentation for properties of `type` and `interface` and
parameters and return values of `function`.

## Usage

### For `type` and `interface`

It generates properties table that displays:

- **name**
- **type**
- **description**
- **default value**
- **permalink**

### For `function`

It generates parameters signature table that displays:

- **name**
- **type**
- **description**
- **default value**.
- **permalink**

together with return signature table that displays:

- **return description**
- **return values table**

> [!TIP]
>
> Permalink is a `#` anchor link to the table row for easy reference.
>
> Descriptions are parsed from [TSDoc](https://tsdoc.org) inline description or
> the `@description` tag. You can use any Markdown/MDX syntax here.
>
> Default values are extracted from the `@default` or `@defaultValue` tag.
>
> Return signature descriptions are extracted from `@returns` tag.

## Example

Props table for `TSDoc` component on this page is generated with following code:

```mdx
import { generateDocumentation, TSDoc } from 'nextra/tsdoc'

<TSDoc
  definitions={generateDocumentation({
    code: `
import type { TSDoc } from 'nextra/tsdoc'
type MyProps = React.ComponentProps<typeof TSDoc>
export default MyProps`
  })}
/>
```

Function signature for `generateDocumentation` function on this page is
generated with following code:

```mdx
import { generateDocumentation, TSDoc } from 'nextra/tsdoc'

<TSDoc
  definitions={generateDocumentation({
    code: "export type { generateDocumentation as default } from 'nextra/tsdoc'"
  })}
/>
```

## Overriding a type

Overridden type can be specified inside the `@remarks` tag using backticks (`).

```ts
type Example = {
  /**
   * **Foo** description.
   * @default "dimaMachina"
   * @remarks `"not dimaMachina"`
   */
  foo: string
}
```

This means that while the property `foo` type is `string`, an alternative type
annotation is described in `@remarks`.
